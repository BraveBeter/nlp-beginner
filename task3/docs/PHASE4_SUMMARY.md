## 📊 关键发现总结

### 训练过程分析
**Transformer vs GPT**:
- **收敛速度**: Transformer 更快 (9 vs 20 epochs 达到最佳)
- **最终性能**: Transformer 略优 (42.21% vs 39.68%)
- **训练效率**: GPT 更快 (0.33s vs 0.43s per epoch)

### 消融实验结果

#### 1. 数据格式影响 🏆
**最有效的改进**:
```
原始格式:  1.56% 序列准确率
反转格式:  3.12% 序列准确率
提升:      +100% ✓
```

**原因分析**:
- 加法从右到左计算（个位→十位→百位）
- 反转后符合 Transformer 的注意力模式
- 编码器更容易学习局部依赖关系

#### 2. 模型容量影响 ❌
**增加参数量无效**:
```
小模型 (5.5M):  1.56% 序列准确率
大模型 (44M):   1.56% 序列准确率
差异:          0%
```

**原因分析**:
- 361 个训练样本不足以训练 44M 参数
- 严重过拟合（训练集 100%，测试集 1.56%）
- **数据量 >> 模型大小** 是关键

#### 3. Scratchpad 方法 ❌
**Token 准确率提升，序列准确率下降**:
```
标准:     Token 32.91%, Sequence 1.56%
Scratchpad: Token 65.00%, Sequence 0.00%
```

**原因分析**:
- 模型学会了格式，但没有学会推理
- 需要更多数据和特定训练策略
- 在小数据集上反而有害
