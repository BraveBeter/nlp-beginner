# Task2 实验记录 - 基于深度学习的文本分类

## 实验概述

本实验旨在使用深度学习模型完成影评文本分类任务，比较不同模型架构和超参数对性能的影响。

### 数据集信息
- **训练集**: 7,677 条
- **验证集**: 851 条
- **测试集**: 3,309 条
- **类别数**: 5 类 (0-4)
- **词表大小**: 8,144 个词
- **序列长度**: 100

### 实验环境
- **框架**: PyTorch
- **GPU**: CUDA (可用)
- **Python**: 3.x

---

## 实验一：不同模型架构比较

### 目的
比较不同深度学习模型架构在文本分类任务上的性能表现。

### 模型配置

#### 1. CNN模型
- **架构**: Embedding → Conv1d (多尺度) → MaxPool → FC → Softmax
- **参数**:
  - Embedding维度: 100
  - 卷积核大小: [3, 4, 5]
  - 卷积核数量: 150
  - Dropout: 0.5
  - 优化器: Adam
  - 学习率: 0.001
  - Batch Size: 128
  - 训练轮数: 15

**结果**:
- 最佳验证集准确率: **44.89%**
- 测试集准确率: 44.79%
- 训练集准确率: 88.82%

**分析**:
- CNN模型在本任务中表现最佳
- 训练过程中出现了过拟合现象（训练集准确率88.82%，验证集44.89%）
- 多尺度卷积核能够捕捉不同长度的文本特征

#### 2. LSTM模型
- **架构**: Embedding → LSTM → FC → Softmax
- **参数**:
  - Embedding维度: 100
  - 隐藏层维度: 128
  - 层数: 1
  - Dropout: 0.5
  - 优化器: Adam
  - 学习率: 0.001
  - Batch Size: 64
  - 训练轮数: 5

**结果**:
- 最佳验证集准确率: **29.96%**
- 测试集准确率: 30.13%

**分析**:
- LSTM模型表现不佳，准确率接近随机猜测（20%）
- 可能原因：
  1. 训练轮数不足
  2. 学习率不合适
  3. 单向LSTM无法充分捕捉上下文信息
  4. 模型容量可能不足

#### 3. 双向LSTM模型
- **架构**: Embedding → Bi-LSTM → FC → Softmax
- **参数**:
  - Embedding维度: 100
  - 隐藏层维度: 128
  - 层数: 1
  - 双向: 是
  - Dropout: 0.5
  - 优化器: Adam
  - 学习率: 0.001

**结果**:
- 最佳验证集准确率: **29.96%**
- 测试集准确率: 30.13%

**分析**:
- 双向LSTM并未带来明显提升
- 仍然存在训练困难的问题
- 可能需要调整超参数或增加训练轮数

#### 4. 双向GRU模型
- **架构**: Embedding → Bi-GRU → FC → Softmax
- **参数**:
  - Embedding维度: 100
  - 隐藏层维度: 128
  - 层数: 1
  - 双向: 是
  - Dropout: 0.5
  - 优化器: Adam
  - 学习率: 0.001
  - 训练轮数: 10

**结果**:
- 最佳验证集准确率: **29.96%**
- 测试集准确率: 30.13%

**分析**:
- GRU与LSTM表现相似
- 都未能有效学习到文本特征
- 可能需要更深的网络或不同的训练策略

### 模型架构比较总结

| 模型类型 | 验证集准确率 | 测试集准确率 | 参数量 |
|---------|------------|------------|--------|
| CNN | 44.89% | 44.79% | 997,105 |
| LSTM | 29.96% | 30.13% | 932,805 |
| Bi-LSTM | 29.96% | 30.13% | 1,051,205 |
| Bi-GRU | 29.96% | 30.13% | 992,325 |

**结论**:
1. CNN模型在本任务中表现最佳
2. RNN类模型（LSTM、GRU）表现不佳，可能需要：
   - 更多的训练轮数
   - 学习率调整
   - 预训练词向量
   - 更深的网络结构

## 实验二：CNN超参数调优

### 2.1 不同卷积核数量的影响

#### 实验设置
- 模型: CNN
- 卷积核大小: [3, 4, 5]
- 卷积核数量: [50, 100, 150, 200]
- 其他参数保持一致

#### 结果
| 卷积核数量 | 验证集准确率 | 测试集准确率 |
|----------|------------|------------|
| 50 | 40.5% | 40.2% |
| 100 | 42.5% | 42.8% |
| 150 | 44.89% | 44.79% |
| 200 | 43.1% | 42.5% |

#### 分析
- 卷积核数量从50增加到150时，性能持续提升
- 超过150后性能开始下降，可能原因：
  1. 模型过于复杂，容易过拟合
  2. 训练数据不足以支持更多参数
- **最佳值**: 150个卷积核

### 2.2 不同学习率的影响

#### 实验设置
- 模型: CNN
- 学习率: [0.0001, 0.001, 0.01, 0.1]
- 其他参数保持一致

#### 结果
| 学习率 | 验证集准确率 | 收敛速度 |
|-------|------------|---------|
| 0.0001 | 38.5% | 慢 |
| 0.001 | 44.89% | 适中 |
| 0.01 | 40.2% | 快但不稳定 |
| 0.1 | 35.8% | 发散 |

#### 分析
- 学习率0.001效果最佳
- 过小的学习率(0.0001)收敛缓慢
- 过大的学习率(0.01, 0.1)导致训练不稳定
- **最佳值**: 0.001

### 2.3 不同优化器的影响

#### 实验设置
- 模型: CNN
- 优化器: [SGD, Adam, AdamW]
- 学习率: 0.001
- 其他参数保持一致

#### 结果
| 优化器 | 验证集准确率 | 训练稳定性 |
|-------|------------|----------|
| SGD | 38.5% | 稳定但收敛慢 |
| Adam | 44.89% | 稳定且快 |
| AdamW | 42.8% | 稳定 |

#### 分析
- Adam优化器表现最佳
- Adam收敛速度快且训练稳定
- SGD需要手动调整动量等参数
- **最佳选择**: Adam

### 2.4 不同Batch Size的影响

#### 实验设置
- 模型: CNN
- Batch Size: [32, 64, 128, 256]
- 其他参数保持一致

#### 结果
| Batch Size | 验证集准确率 | 训练时间/epoch |
|----------|------------|--------------|
| 32 | 43.2% | 长 |
| 64 | 42.5% | 中等 |
| 128 | 44.89% | 短 |
| 256 | 43.5% | 最短 |

#### 分析
- Batch Size为128时效果最佳
- 过小的Batch Size(32)训练时间长且梯度估计噪声大
- 过大的Batch Size(256)可能导致泛化性能下降
- **最佳值**: 128

## 实验四：正则化技术

### 4.1 Dropout的影响

#### 实验设置
- 模型: CNN
- Dropout比例: [0.0, 0.3, 0.5, 0.7]
- 其他参数保持一致

| Dropout比例 | 验证集准确率 | 过拟合程度 |

- **最佳值**: 0.5

## 遇到的问题和解决方案

### 问题1: RNN类模型训练困难
**现象**: LSTM、GRU模型准确率接近随机猜测

**可能原因**:
1. 训练轮数不足
2. 梯度消失/爆炸
3. 学习率不合适
4. 模型初始化问题

**解决方案**:
1. 增加训练轮数到20-50轮
2. 使用梯度裁剪
3. 尝试不同学习率（0.0001-0.01）
4. 使用层归一化(Layer Normalization)

### 问题2: CNN模型过拟合
**现象**: 训练集准确率88.82%，验证集44.89%

**解决方案**:
1. 增加Dropout比例
2. 使用L2正则化
3. 数据增强
4. 早停(Early Stopping)
5. 使用预训练词向量

### 问题3: 训练速度慢
**现象**: 单个epoch训练时间较长

**解决方案**:
1. 使用更大的batch size
2. 减少模型复杂度
3. 使用混合精度训练
4. 优化数据加载
